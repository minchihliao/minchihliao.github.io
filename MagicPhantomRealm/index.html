Project Overview:

Phantom Realms is a mobile game designed for players aged 15 and above, featuring a layer-based adventure structure reminiscent of Slay the Spire. Players enhance their character's abilities through unique events to triumph over various monsters.

Technical Framework:

  • Development Tools: Built in Unity, integrating multiple third-party databases for enhanced data management.
  • Asynchronous Operations: Utilized UniTask for handling asynchronous events such as data downloads and animation sequences, ensuring smooth gameplay and interface responsiveness.
  • Reactive Programming: Implemented UniRx for dynamic data observation, allowing for real-time UI updates based on user data changes.
  • Architecture: Employed a Finite State Machine (FSM) for clearer and more manageable game flow development, alongside MVC architecture to optimize UI control and application performance.

    • My Role and Contributions:

  • Event and Gameplay Programming:
      Developed features across several game systems including class selection, level progression, and rewards. Implemented skills and item purchasing mechanics, integrating player data with server-side communications.
  • UI/UX Design and Animation:
      Applied MVC architecture principles to design a clean, modular UI script structure that separates data access, control, and display. Utilized DoTween and UniTask for efficient animations.
  • Integration of Third-Party Tools:
      Used SDK functions for mock server interactions and Google Sheets for item data retrieval and updates, enhancing game performance and scalability.
  • Cross-functional Collaboration:
      Collaborated with artists and UI designers to ensure a cohesive game aesthetic, aligning technical functionality with artistic vision.

    • Challenges and Solutions:

  • Efficiently Structuring Game UI
      Problem
        The initial UI was cumbersome and made it difficult for players to navigate through various game features like skill selection and skill level up.
      Solution
        Applied the MVC architectural pattern to redesign the UI, creating a clear separation between data, view, and controller. This reorganization allowed for more modular and maintainable code, making future updates easier to implement.
      Impact
        The new UI design improved user engagement and made the interface more intuitive, which helped in increasing the player retention rate and satisfaction.
  • Efficient Data Retrieval from Google Sheets for Item Management
      Problem
        A key challenge in the game development was ensuring efficient retrieval and integration of item data from Google Sheets into the game, which was crucial for the item acquisition feature.
      Solution
      • Data Retrieval Implementation: Collaborated in the integration process by specifying how item data should be retrieved from Google Sheets. This involved defining the data access methods and ensuring that the retrieved data was correctly formatted and synchronized with the game's existing item management system.
      • Item Data Structure Definition: Worked on defining the data structure that stores information about player-acquired items in the game, ensuring it supported efficient access and manipulation during gameplay.
      • Mock Server Utilization: Assisted in setting up and defining how a mock server would simulate the retrieval and storage of item data, crucial for testing the system's robustness and response under various scenarios.
      Impact
      • The precise definition of data retrieval methods from Google Sheets enabled seamless and error-free updates to item data, enhancing the game's ability to introduce new items without extensive downtime or manual intervention.
      • The well-defined item data structure ensured that players experienced minimal latency and high reliability when accessing and using items in the game, leading to improved gameplay satisfaction.
      • The use of a mock server for testing the integration allowed the development team to identify and address potential issues early in the development cycle, reducing the risk of bugs and ensuring a smooth rollout of the item acquisition feature.
